# Cilium + Hubble — Replace kube-proxy, kube-router, Envoy Gateway, MetalLB

> **For Hermes:** Use subagent-driven-development to implement this plan task-by-task.

**Goal:** Implement Cilium as the cluster CNI with eBPF kube-proxy replacement, BGP load balancing, Gateway API, and Hubble observability — replacing four networking components.

**What gets replaced:**
- kube-router (CNI) → Cilium eBPF CNI
- kube-proxy (IPVS) → Cilium eBPF kube-proxy replacement
- Envoy Gateway → Cilium Gateway API
- MetalLB (BGP speaker) → Cilium BGP control plane

**Repository:** zacheryph/k8s-gitops

**Current state (cluster probe 2026-07-15):**
- 3 nodes: k0s-inwin-01/02/03 (k8s v1.36.2+k0s), all controller+worker
- 6 LoadBalancer services using 10.72.16.x pool (MetalLB BGP → OPNsense)
- Envoy Gateway serving ~20 HTTPRoutes via Gateway `external` in namespace `gateway`
- ExternalDNS (AdGuard + Cloudflare) consuming `gateway-httproute` sources — no changes needed
- TLS certs via cert-manager ClusterIssuer `letsencrypt` (DNS-01 Cloudflare) — no changes needed
- CoreDNS in kube-system (k0s-managed) — no changes needed
- SOPS vars: ROUTER_IP, ROUTER_ASN, CLUSTER_ASN, LOAD_BALANCER_POOL, LOAD_BALANCER_PLEX, CLUSTER_DOMAIN, WEB_DOMAIN, CLOUD_EXTERNAL_IP

**Architecture:**

```
                    ┌─────────────────────────────────┐
                    │         Cilium (core/)            │
                    ├─────────────────────────────────┤
                    │ eBPF CNI (replaces kube-router)  │
                    │ eBPF kube-proxy replacement       │
                    │ BGP control plane (→ OPNsense)    │
                    │ Gateway API (replaces Envoy GW)   │
                    │ Hubble + Hubble Relay + UI        │
                    │ CiliumLoadBalancerIPPool (IPAM)   │
                    └──────────────┬────────────────────┘
                                   │
              ┌────────────────────┼────────────────────┐
              │                    │                     │
     CiliumBGPClusterConfig  CiliumBGPPeerConfig  CiliumBGPAdvertisement
     (core/resources/)        (core/resources/)     (core/resources/)
     localASN, peerAddress    timers, transport     Service type advertisement
              │                    │                     │
              └────────────────────┴─────────────────────┘
                                   │
                          BGP session to OPNsense
                          (same ASN/peer as MetalLB)
```

**Tech stack:**
- Cilium 1.19.5 (Helm chart via OCI: `oci://ghcr.io/cilium/charts/cilium`)
- Cilium BGP CRDs: `cilium.io/v2` (CiliumBGPClusterConfig, CiliumBGPPeerConfig, CiliumBGPAdvertisement)
- CiliumLoadBalancerIPPool: `cilium.io/v2` (replaces MetalLB IPAddressPool)
- Gateway API: `gateway.networking.k8s.io/v1` (Cilium GatewayClass `cilium`)

**Assumptions:**
- Cilium CRDs are installed by the cilium-operator at startup (the Helm chart does NOT include CRD manifests). For initial install, the operator creates CRDs on first run. For GitOps, the CRD resources (BGP/LBIPPool) in `core/resources/` will be applied after the HelmRelease — if CRDs aren't ready yet on first reconcile, Flux will retry.
- k0s `spec.network.provider: custom` opts out of k0s-managed CNI. Cilium's DaemonSet installs the CNI binary and config on each node.
- `spec.network.kubeProxy.disabled: true` removes kube-proxy entirely; Cilium's `kubeProxyReplacement: true` takes over.
- `nodeLocalLoadBalancing` (EnvoyProxy NLLB) must be removed — it conflicts with Cilium's own kube-proxy replacement and service handling.
- `strictARP: true` was for MetalLB+FRR; Cilium BGP handles ARP differently (no strictARP needed).
- Existing HTTPRoutes reference `parentRefs: [{name: external, namespace: gateway}]` — Cilium Gateway API supports the same Gateway resource shape, so HTTPRoutes need NO changes. Only the GatewayClass changes from `envoy` to `cilium`.

**Migration path (documented, not executed in this PR — Draft PR):**
1. Apply Cilium HelmRelease + CRDs (Flux reconciles, Cilium DaemonSet starts alongside kube-router)
2. Verify Cilium agents are Ready on all nodes
3. Drain nodes one at a time: `kubectl drain <node> --ignore-daemonsets --delete-emptydir-data`
4. On each drained node: `k0sctl apply --config config/cluster.yaml` (switches provider to custom, disables kube-proxy, removes NLLB)
5. Uncordon node, verify Cilium CNI is active, pods get IPs
6. Repeat for all 3 nodes
7. Remove kube-router DaemonSet manually (prune:false means Git won't remove it)
8. Remove MetalLB + Envoy Gateway HelmReleases (this PR annotates them for removal)
9. Verify all HTTPRoutes work through Cilium Gateway API
10. Verify BGP peering to OPNsense and LB IPs are advertised

### Task 1: Create Cilium HelmRelease + OCIRepository

**Objective:** Add the Cilium Helm chart as an OCIRepository and HelmRelease in `core/`.

**File:** `core/cilium.yaml`

```yaml
# yaml-language-server: $schema=https://kubernetes-schemas.pages.dev/source.toolkit.fluxcd.io/ocirepository_v1.json
---
apiVersion: source.toolkit.fluxcd.io/v1
kind: OCIRepository
metadata:
  name: cilium
  namespace: kube-system
spec:
  interval: 12h
  layerSelector:
    mediaType: application/vnd.cncf.helm.chart.content.v1.tar+gzip
    operation: copy
  ref:
    tag: 1.19.5
  url: oci://ghcr.io/cilium/charts/cilium
---
# yaml-language-server: $schema=https://kubernetes-schemas.pages.dev/helm.toolkit.fluxcd.io/helmrelease_v2.json
apiVersion: helm.toolkit.fluxcd.io/v2
kind: HelmRelease
metadata:
  name: cilium
  namespace: kube-system
spec:
  chartRef:
    kind: OCIRepository
    name: cilium
  interval: 1h
  upgrade:
    crds: CreateReplace
  values:
    # eBPF kube-proxy replacement
    kubeProxyReplacement: true
    kubeProxyReplacementHealthzBindAddr: "0.0.0.0:10256"

    # BGP control plane
    bgpControlPlane:
      enabled: true
      statusReport:
        enabled: true

    # Gateway API (replaces Envoy Gateway)
    gatewayAPI:
      enabled: true
      gatewayClass:
        create: true
      externalTrafficPolicy: Local
      secretsNamespace:
        create: true
        name: cilium-secrets
        sync: true

    # Hubble observability
    hubble:
      enabled: true
      relay:
        enabled: true
      ui:
        enabled: true
      metrics:
        enabled:
          - dns
          - drop
          - tcp
          - flow
          - icmp

    # Prometheus metrics
    prometheus:
      enabled: true
      serviceMonitor:
        enabled: true
        trustCRDsExist: true

    # Operator metrics
    operator:
      prometheus:
        enabled: true
        serviceMonitor:
          enabled: true
          trustCRDsExist: true

    # Hubble Relay/Operator metrics
    hubble:
      relay:
        prometheus:
          enabled: true
          serviceMonitor:
            enabled: true
            trustCRDsExist: true

    # L2 announcements disabled — using BGP for LB IP advertisement
    l2announcements:
      enabled: false

    # IPAM mode
    ipam:
      mode: cluster-pool

    # Security context — Cilium needs privileged for eBPF
    securityContext:
      capabilities:
        ciliumAgent:
          - CHOWN
          - KILL
          - NET_ADMIN
          - NET_RAW
          - IPC_LOCK
          - SYS_ADMIN
          - SYS_RESOURCE
          - DAC_OVERRIDE
          - FOWNER
          - SETGID
          - SETUID
        cleanCiliumAgent:
          - NET_ADMIN
          - SYS_ADMIN
          - SYS_RESOURCE
          - DAC_OVERRIDE
          - FOWNER
          - SETGID
          - SETUID
        mountCgroup:
          - SYS_ADMIN
        applySysctl:
          - SYS_ADMIN
        mountBpfFs:
          - SYS_ADMIN
        mountTlsFiles:
          - SYS_ADMIN

    # Node selector — all nodes
    nodeSelector:
      kubernetes.io/os: linux

    # Tolerate all nodes (control plane + worker)
    tolerations:
      - operator: Exists
```

**Commit:**
```bash
git add core/cilium.yaml
git commit -m "feat(core): add Cilium HelmRelease with eBPF, BGP, Gateway API, Hubble"
```

### Task 2: Create Cilium BGP + LoadBalancer IPPool resources

**Objective:** Add Cilium BGP configuration (replacing MetalLB IPAddressPool + BGPPeer + BGPAdvertisement) and CiliumLoadBalancerIPPool for IPAM.

**File:** `core/resources/cilium-bgp.yaml`

```yaml
# yaml-language-server: $schema=https://kubernetes-schemas.pages.dev/cilium.io/ciliumbgpclusterconfig_v2.json
---
apiVersion: cilium.io/v2
kind: CiliumBGPClusterConfig
metadata:
  name: router
spec:
  nodeSelector:
    matchLabels:
      kubernetes.io/os: linux
  bgpInstances:
    - name: router
      localASN: ${CLUSTER_ASN}
      peers:
        - name: router
          peerAddress: ${ROUTER_IP}
          peerASN: ${ROUTER_ASN}
          peerConfigRef:
            name: router
---
# yaml-language-server: $schema=https://kubernetes-schemas.pages.dev/cilium.io/ciliumbgppeerconfig_v2.json
apiVersion: cilium.io/v2
kind: CiliumBGPPeerConfig
metadata:
  name: router
spec:
  transport:
    peerPort: 179
  timers:
    holdTimeSeconds: 90
    keepAliveTimeSeconds: 30
  families:
    - afi: ipv4
      safi: unicast
      advertisements:
        matchLabels:
          bgp.routine.sh/advertise: "true"
---
# yaml-language-server: $schema=https://kubernetes-schemas.pages.dev/cilium.io/ciliumbgpadvertisement_v2.json
apiVersion: cilium.io/v2
kind: CiliumBGPAdvertisement
metadata:
  name: services
  labels:
    bgp.routine.sh/advertise: "true"
spec:
  advertisements:
    - advertisementType: Service
      service:
        addresses:
          - LoadBalancerIP
        aggregationLengthIPv4: 32
---
# yaml-language-server: $schema=https://kubernetes-schemas.pages.dev/cilium.io/ciliumloadbalancerippool_v2.json
apiVersion: cilium.io/v2
kind: CiliumLoadBalancerIPPool
metadata:
  name: services-block
spec:
  allowFirstLastIPs: "No"
  blocks:
    - cidr: ${LOAD_BALANCER_POOL}
```

### Task 8: Add Hubble UI HTTPRoute

**Objective:** Add an HTTPRoute for Hubble UI so it's accessible via the cluster's Gateway API.

**File:** `platform/gateway/hubble-ui.yaml`

```yaml
# yaml-language-server: $schema=https://kubernetes-schemas.pages.dev/gateway.networking.k8s.io/httproute_v1.json
---
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: hubble-ui
  namespace: kube-system
  annotations:
    dns.routine.sh/external: "true"
spec:
  parentRefs:
    - name: external
      namespace: gateway
  hostnames:
    - "hubble.${CLUSTER_DOMAIN}"
  rules:
    - matches:
        - path:
            type: PathPrefix
            value: /
      backendRefs:
        - name: hubble-ui
          port: 80
```

**File:** `platform/gateway/kustomization.yaml` — add `- hubble-ui.yaml` to resources.
